🔐 Passive vs Active Reconnaissance

Reconnaissance is the first phase of ethical hacking, divided into two main approaches:

🟢 Passive Reconnaissance

  • Gathers information without directly interacting with the target.
  • Uses public sources: search engines, WHOIS, DNS records, social media, job postings.
  • Low risk of detection — the target never knows you're looking.

🔴 Active Reconnaissance

  • Involves direct interaction with the target's systems.
  • Examples: port scanning, DNS zone transfers, banner grabbing, ping sweeps.
  • Higher risk — may trigger alerts or be logged by the target.

Golden Rule: Always obtain written authorization before any active reconnaissance!

📊 Information Gathering Categories

  • Network Information: IP ranges, subdomains, DNS records, network topology.
  • System Information: Operating systems, open ports, running services.
  • Organizational Information: Employee names, emails, phone numbers, physical locations.
  • Technical Footprint: Web technologies, CMS, server types, security headers.

🔎 Google Dorking (Google Hacking)

Using advanced search operators to find information that wasn't meant to be public.

Common Operators:
site:example.com
intitle:"index of"
filetype:pdf confidential
inurl:admin
Example Dorks:
site:example.com filetype:sql
intitle:"webcam" inurl:view
"password" filetype:xls
cache:example.com

Google Dorking is a powerful OSINT technique. Use responsibly and ethically.

🧠 Understanding WHOIS Output

  • Registrar: Company where the domain was registered (e.g., GoDaddy, Namecheap).
  • Creation/Expiration Dates: Useful for social engineering or understanding domain age.
  • Name Servers: Reveals hosting provider and potential subdomains.
  • Registrant Contact: Often redacted (GDPR), but older domains may show email/phone.
