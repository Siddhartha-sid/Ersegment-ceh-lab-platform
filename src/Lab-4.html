📡 Network Scanning Overview

Network scanning is the second phase of ethical hacking. It involves identifying live hosts, open ports, running services, and operating systems.

Objectives:

  • Discover live hosts on a network (ICMP, ARP)
  • Identify open ports and services (TCP/UDP)
  • Detect operating systems (fingerprinting)
  • Banner grabbing for service versions
  • Evade firewalls and IDS

⚙️ Common Scanning Techniques

🔹 TCP Connect Scan (-sT) — Completes full 3-way handshake. Noisy but reliable.
🔹 SYN Stealth Scan (-sS) — Sends SYN, receives SYN-ACK, sends RST. Default for Nmap.
🔹 UDP Scan (-sU) — Sends UDP packets, looks for ICMP unreachable or response.
🔹 OS Detection (-O) — Fingerprints OS via TCP/IP stack behavior.

🛡️ Evasion Techniques

  • Fragment packets (-f)
  • Use decoys (-D)
  • Spoof source IP/port (-S, -g)
  • Timing options (-T0 to -T5)
  • Randomize scan order (--randomize-hosts)

📊 Interpreting Scan Results

Port States:

  • Open — Service is listening.
  • Filtered — Firewall blocking probe.
  • Closed — No service, but host reachable.

Banner Grabbing:

Use netcat or nmap -sV to retrieve service banners. Example: nc scanme.local 22 might reveal SSH version.
